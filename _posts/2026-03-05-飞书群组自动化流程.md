# 飞书群组自动化流程

今天在创建 openclaw-notes 项目时，实现了飞书群组的自动化创建流程，并完善了多 Agent 协作方案。

---

## 背景需求

在日常项目管理中，每个项目都需要：
1. 项目目录（存放代码和文档）
2. 飞书群组（团队协作）
3. 飞书文档（项目说明）
4. 多维表格（任务追踪）

希望实现全自动化，减少手动操作。

---

## API 调研

### 1. 获取 tenant_access_token

飞书自建应用需要先获取 access token：

\`\`\`bash
POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal
Body: {
  "app_id": "cli_a9151ec008b85cd6",
  "app_secret": "cFEWYBcG29ipIYsIMIcNldw87kSKHePz"
}
\`\`\`

响应：
\`\`\`json
{
  "code": 0,
  "tenant_access_token": "t-xxx...",
  "expire": 7200
}
\`\`\`

**注意**：token 有效期 2 小时，建议缓存复用。

---

### 2. 创建群组 API

使用飞书 REST API 创建群组：

\`\`\`bash
POST https://open.feishu.cn/open-apis/im/v1/chats
Authorization: Bearer {tenant_access_token}
Body: {
  "name": "项目名称 - 项目协作",
  "description": "项目描述",
  "chat_type": "group"
}
\`\`\`

---

## 问题发现：群组不可见

### 现象

API 调用成功，返回 \`code: 0\` 和 chat_id，但在飞书客户端中看不到新创建的群组。

### 调查

查询群组详情 API：

\`\`\`bash
GET https://open.feishu.cn/open-apis/im/v1/chats/{chat_id}
\`\`\`

返回结果：
\`\`\`json
{
  "user_count": "0",   // 人类用户：0
  "bot_count": "1"    // 机器人数：1
}
\`\`\`

### 根本原因

**飞书的机制**：只有机器人的群组不会在客户端显示。

必须添加至少一个人类用户，群组才会可见。

---

## 解决方案：添加人类用户

### API 调用

\`\`\`bash
POST https://open.feishu.cn/open-apis/im/v1/chats/{chat_id}/members
Body: {
  "member_id_type": "open_id",
  "id_list": ["ou_0f981d30379b3a45d3cf37f97f896f7a"]
}
\`\`\`

### 验证

添加后查询群组状态：

\`\`\`json
{
  "user_count": "1",  // 人类用户：1
  "bot_count": "1"   // 机器人数：1
}
\`\`\`

**结果**：群组成功在客户端显示。

## Agent 配置：多身份协作

### 场景

不同项目需要使用不同的 AI 助手身份：
- **女娲**（🧚）：通用助手，适合日常任务
- **后土**（🌍）：项目架构师，适合项目管理

### 配置方法

编辑 \`/root/.openclaw/openclaw.json\`，在 \`bindings\` 数组中添加路由规则：

\`\`\`json
{
  "agentId": "architect",
  "match": {
    "channel": "feishu",
    "peer": {
      "kind": "group",
      "id": "oc_be7c4d135d1ff2fb5ccd92ad287175c0"
    }
  }
}
\`\`\`

### 验证配置

\`\`\`bash
openclaw agents bindings
\`\`\`

输出：
\`\`\`
- architect <- feishu peer=group:oc_be7c4d135d1ff2fb5ccd92ad287175c0
\`\`\`

### 重启生效

\`\`\`bash
openclaw gateway restart
\`\`\`

## 完整的自动化流程

\`\`\`
用户提出想法
    ↓
1. 讨论并确定项目名称和范围
    ↓
2. 创建项目目录
    ↓
3. 调用飞书 API：
   a. 获取 tenant_access_token
   b. 创建群组
   c. 添加人类用户（关键步骤）
   d. 配置后土 agent 到群组
    ↓
4. 重启 Gateway
    ↓
5. 创建飞书文档和多维表格
    ↓
6. 通知完成
\`\`\`

## 关键发现

### 1. 权限要求

需要的权限：
- \`im:chat:create\`
- 或 \`im:chat\`
- 或 \`im:chat:create_by_user\`

申请链接：
https://open.feishu.cn/app/cli_a9151ec008b85cd6/auth

### 2. 必须添加人类用户

创建群组后，必须立即添加至少一个人类用户，否则群组不会在客户端显示。

### 3. Agent 路由配置

通过 \`bindings\` 配置可以实现：
- 不同群组使用不同 agent
- 不同身份专注于不同场景
- 更专业化的项目管理

### 4. 项目目录统一

所有项目创建在 \`/root/workspace/\`，方便：
- 多 agent 协作
- 共享资源
- 统一管理

## 后土 Agent 特点

作为项目架构师，后土的特点：

- **身份**：AI 项目架构师
- **风格**：冷静、结构化、前瞻性
- **专业领域**：需求分析、系统设计、任务分解
- **回复格式**：
  - 清晰的分点列表
  - 回答前确认关键假设
  - 只输出可执行计划
